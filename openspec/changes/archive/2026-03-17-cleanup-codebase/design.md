## Context

项目已完成 MVP 阶段，包含：
- 核心消息系统（统一消息结构、路由、会话、配置）
- 飞书平台适配器（WebSocket 长连接、消息收发）
- Claude Code 适配器（子进程管理、流式输出、权限模式）
- 斜杠命令系统

当前代码库状态良好：
- 测试全部通过
- `go vet` 无警告
- 无 TODO/FIXME 遗留
- 测试覆盖率达标（核心功能 >85%）

存在的历史遗留问题：
1. 早期规划文档 `docs/Go 语言 TDD 实现 cc-connect 完整方案（含功能详解）.md` 已过时
2. 测试文件 `coverage_test.go` 命名不够清晰

## Goals / Non-Goals

**Goals:**
- 归档过时的规划文档，保留历史参考价值
- 重命名测试文件为更具描述性的名称
- 确保项目文档与当前实现状态一致

**Non-Goals:**
- 不修改任何运行时代码
- 不修改测试逻辑或覆盖率
- 不删除任何有用文档
- 不引入新功能

## Decisions

### D1: 归档策略

**决定**：创建 `docs/archive/` 目录，将过时文档移动至此并添加归档标记。

**理由**：
- 保留历史参考价值
- 明确区分当前有效文档和归档文档
- 便于未来查阅项目演进历史

**替代方案**：
- 直接删除：❌ 丢失历史信息
- 重命名为 `.archived.md`：❌ 污染 docs 目录

### D2: 测试文件重命名

**决定**：将 `coverage_test.go` 重命名为 `edge_cases_test.go`

**理由**：
- `edge_cases_test.go` 更准确描述测试内容（边缘情况测试）
- 符合 Go 测试命名惯例
- 便于开发者理解文件用途

**替代方案**：
- 保留原名：❌ 名称不直观
- 拆分为多个文件：❌ 不必要的复杂度

## Risks / Trade-offs

### R1: 文件移动影响 Git 历史

**风险**：移动文件后，`git blame` 追踪需要额外步骤

**缓解**：
- Git 会自动追踪文件重命名
- 使用 `git log --follow` 可查看完整历史

### R2: 测试文件重命名可能影响 IDE 书签

**风险**：开发者可能有指向旧文件路径的书签

**缓解**：
- 影响范围小，开发者可自行更新
- 一次性操作，无持续影响
